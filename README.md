# Portal.Bot — Telegram-бот цифровой экосистемы МФТИ

**Бэкенд (Portal2):** https://github.com/korsixe/Portal2/

**Фронтенд:** https://github.com/ArturOzolin/portal-frontend

Portal.Bot — Telegram-бот, который является частью единой цифровой экосистемы Portal для студентов МФТИ. Бот привязывает аккаунт Telegram к аккаунту на портале, позволяет просматривать свои объявления и доставляет push-уведомления о событиях маркетплейса прямо в мессенджер.

Проект реализуется в рамках программы "Высшая школа программной инженерии" МФТИ при поддержке компании ООО MWS.

---

## Архитектура

Бот — самостоятельный Spring Boot-микросервис, подключённый к той же БД, что и основной бэкенд Portal2:

- **Telegram API:** Long Polling через `telegrambots-spring-boot-starter`
- **Бизнес-логика:** Spring Boot 3, обработка команд и callback-запросов
- **Слой данных:** PostgreSQL, общая БД с Portal2 через Spring Data JPA
- **Планировщик:** Spring Scheduling — периодическая отправка уведомлений

## Конфигурация

### application.yml

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5433/myproject
    username: myuser
    password: ${DB_PASSWORD:mypassword}
    driver-class-name: org.postgresql.Driver

  jpa:
    hibernate:
      ddl-auto: update

telegram:
  bot:
    name: "PhystechPortalBot"
    token: "YOUR_BOT_TOKEN"
```

Для локального запуска создай файл `application-local.yml` в `src/main/resources/` и переопредели токен и пароль БД:

```yaml
spring:
  datasource:
    password: mypassword

telegram:
  bot:
    token: "your-bot-token-here"
```

### Подключение к БД

- **URL:** `jdbc:postgresql://localhost:5433/myproject`
- **Пользователь:** `myuser`
- **Пароль:** `mypassword`
- **Драйвер:** `org.postgresql.Driver`

> Бот использует ту же базу данных, что и Portal2. Убедись, что контейнер с PostgreSQL из Portal2 запущен перед стартом бота.

## Запуск проекта

### 1. Клонирование репозитория

```bash
git clone https://github.com/korsixe/Portal.Bot
cd Portal.Bot
```

### 2. Запуск инфраструктуры

Убедись, что PostgreSQL из Portal2 уже поднят:

```bash
cd ../Portal2
docker-compose up -d db
```

### 3. Настройка переменных окружения

Создай файл `src/main/resources/application-local.yml`:

```yaml
telegram:
  bot:
    token: "your-bot-token-from-botfather"
```

Либо передай токен через переменную окружения при запуске.

### 4. Запуск бота

```bash
mvn spring-boot:run
```

Бот начнёт принимать сообщения от пользователей в Telegram.

## Команды бота

| Команда | Описание |
|---|---|
| `/start` | Привязать аккаунт Telegram к аккаунту Portal (по email `@phystech.edu`) |
| `/my_ads` | Показать свои объявления: активные и черновики |
| `/bookings` | Показать активные бронирования с оставшимся временем |
| `/search <запрос>` | Найти объявления по тексту (или `/search` — бот спросит запрос) |
| `/unlink` | Отвязать аккаунт Portal от этого Telegram |

## Workflow

### Привязка аккаунта

1. Пользователь отправляет `/start`
2. Бот запрашивает корпоративный email (`@phystech.edu`)
3. Пользователь вводит email — бот ищет его в БД Portal
4. При успехе аккаунты привязываются, Telegram `chat_id` сохраняется
5. При повторном `/start` бот приветствует по имени

### Просмотр объявлений

1. Пользователь отправляет `/my_ads`
2. Бот выводит все объявления со статусами **Активные** и **Черновики**
3. Для каждого объявления показывается название, цена и дата обновления

### Отвязка аккаунта

1. Пользователь отправляет `/unlink`
2. Бот показывает email привязанного аккаунта и запрашивает подтверждение через inline-кнопки «Да, отвязать» / «Отмена»
3. При подтверждении — `telegramChatId` и `telegramUsername` очищаются в БД, уведомления перестают приходить
4. После отвязки можно заново привязать другой аккаунт через `/start`

### Поиск объявлений

1. Пользователь отправляет `/search велосипед` или просто `/search`
2. Во втором случае бот просит ввести запрос следующим сообщением
3. Бот ищет активные объявления по тексту в названии и описании
4. Возвращает до 5 результатов: название, цена, состояние, категория, местоположение
5. Если найдено больше 5 — сообщает общее количество

### Просмотр бронирований

1. Пользователь отправляет `/bookings`
2. Бот выводит список активных бронирований (не отменённых и не подтверждённых)
3. Для каждого бронирования показывается название товара, цена и оставшееся время до автоотмены (24 ч с момента создания)
4. Если активных бронирований нет — бот сообщает об этом

### Подтверждение актуальности объявления

1. Бот находит объявления, не обновлявшиеся более 30 дней
2. Отправляет уведомление с inline-кнопкой «✅ Всё ещё актуально»
3. Если пользователь не нажал кнопку в течение 24 часов — объявление переводится в архив
4. Если нажал — дата обновления сбрасывается, объявление остаётся активным

## Telegram-уведомления

Бот автоматически отправляет сообщения при следующих событиях:

| Событие | Получатель | Интервал проверки |
|---|---|---|
| Новое бронирование создано | Продавец и покупатель | каждую минуту |
| Бронирование отменено | Продавец и покупатель | каждую минуту |
| Продажа подтверждена | Продавец и покупатель | каждую минуту |
| Объявление не обновлялось 30 дней | Автор объявления | ежедневно в 23:00 |
| Объявление переведено в архив | Автор объявления | ежедневно в 23:00 |

## Особенности реализации

- Привязка только по email МФТИ (`@phystech.edu`)
- Бот работает в режиме Long Polling — не требует публичного HTTPS-сервера
- Планировщик проверяет события бронирования каждую минуту (`fixedDelay = 60s`)
- Проверка устаревших объявлений — раз в сутки в 23:00 (`cron = "0 0 23 * * *"`)
- Уведомления о каждом событии отправляются ровно один раз (флаги `notificationSentAt`, `cancelNotificationSentAt`, `confirmNotificationSentAt`)
- При изменении Telegram-никнейма пользователя бот автоматически обновляет его в БД
